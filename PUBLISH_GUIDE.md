# 发布到 ClawHub 指南

本指南详细说明如何将改造后的 Bole Story Creator 技能发布到 ClawHub 平台。

## 准备工作

1. **确保技能目录结构完整**
   - 检查 `bole-story-creator` 目录是否包含以下文件：
     - `skill.json`：技能配置文件
     - `main.py`：主执行文件
     - `requirements.txt`：依赖文件
     - `README.md`：技能说明文档

2. **准备技能图标（可选）**
   - 可以准备一个 128x128 的 PNG 图标文件，命名为 `icon.png`，放在技能目录中

3. **测试技能功能**
   - 确保技能能够正常运行，输入输出格式正确

## 发布步骤

### 方法一：通过 ClawHub 网页界面发布

1. **访问 ClawHub 平台**
   - 打开浏览器，访问 ClawHub 官方网站（例如：https://clawhub.com）
   - 登录您的账号

2. **创建新技能**
   - 点击 "发布技能" 或 "New Skill" 按钮
   - 填写技能基本信息：
     - 技能名称：Bole Story Creator
     - 版本：1.0.0
     - 描述：调用博乐AI平台生成短剧故事
     - 标签：story, video, creation, bole, ai

3. **上传技能文件**
   - 选择 "上传技能包" 选项
   - 将 `bole-story-creator` 目录压缩为 ZIP 文件
   - 上传 ZIP 文件

4. **配置技能参数**
   - 确认技能配置信息正确
   - 设置技能的访问权限（公开/私有）

5. **提交审核**
   - 点击 "提交审核" 按钮
   - 等待 ClawHub 平台审核通过

### 方法二：通过 OpenClaw CLI 发布

1. **安装 OpenClaw CLI**
   - 确保您已安装 OpenClaw CLI 工具
   - 可以通过 pip 安装：`pip install openclaw-cli`

2. **登录 ClawHub**
   - 运行命令：`openclaw login`
   - 输入您的 ClawHub 账号信息

3. **发布技能**
   - 在技能目录的父目录中运行命令：
     ```bash
     openclaw publish bole-story-creator
     ```
   - 按照提示填写技能信息

4. **确认发布**
   - 等待发布完成
   - 查看发布结果

## 发布后管理

1. **查看技能状态**
   - 登录 ClawHub 平台
   - 在 "我的技能" 中查看技能状态

2. **更新技能**
   - 如需更新技能，修改相关文件后重新发布
   - 版本号应递增

3. **管理技能设置**
   - 可以在 ClawHub 平台上修改技能的设置、描述、标签等信息

## 注意事项

1. **环境变量配置**
   - 使用技能时需要设置 `BOLE_ACCESS_KEY` 环境变量
   - 在 ClawHub 平台上可以设置技能的环境变量

2. **权限设置**
   - 确保技能的权限设置合理，避免敏感信息泄露

3. **技能测试**
   - 发布前务必测试技能功能，确保正常运行

4. **文档更新**
   - 及时更新技能文档，确保用户能够正确使用技能

## 故障排除

1. **发布失败**
   - 检查技能目录结构是否完整
   - 确保 `skill.json` 格式正确
   - 检查网络连接

2. **技能运行失败**
   - 检查 `BOLE_ACCESS_KEY` 是否正确设置
   - 查看技能运行日志，定位错误原因
   - 确保博乐AI平台 API 正常可用

3. **审核未通过**
   - 检查技能是否符合 ClawHub 的发布规范
   - 完善技能文档和描述
   - 确保技能功能合法合规

---

通过以上步骤，您可以成功将 Bole Story Creator 技能发布到 ClawHub 平台，让更多用户使用这个技能来生成短剧故事。
